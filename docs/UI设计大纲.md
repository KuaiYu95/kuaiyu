# UI 设计大纲

## 一、设计理念

### 1.1 设计风格

- **极简暗黑风格**: 深色背景，高对比度，减少视觉干扰
- **居中布局**: 所有内容居中展示，最大宽度 800px
- **丝滑动效**: 使用现代 CSS 和 JS 动画库，提升交互体验
- **字体规范**: 默认 14px，层次分明

### 1.2 色彩方案

#### 主色调（暗黑主题）

```
背景色:
- 主背景: #0a0a0a (纯黑)
- 卡片背景: #1a1a1a (深灰)
- 悬浮背景: #252525 (中灰)

文字色:
- 主文字: #e5e5e5 (浅灰)
- 次要文字: #a0a0a0 (中灰)
- 强调文字: #ffffff (纯白)
- 链接文字: #60a5fa (蓝色)

强调色:
- 主强调: #60a5fa (蓝色)
- 次要强调: #34d399 (绿色)
- 警告: #fbbf24 (黄色)
- 错误: #f87171 (红色)

边框色:
- 默认边框: #2a2a2a
- 悬浮边框: #3a3a3a
```

#### 代码高亮主题

- 使用暗色主题，如: One Dark Pro, Dracula, Monokai

## 三、动效设计

### 3.1 页面滚动动效

1. **视差滚动**

   - 背景元素缓慢移动
   - 文字内容正常滚动

2. **元素渐入**

   - 滚动到视口时，元素从下方淡入+上移
   - 使用 Intersection Observer API

3. **滚动进度条**
   - 顶部显示阅读进度条
   - 平滑动画

### 3.2 Hover 动效

1. **卡片悬浮**

   - 轻微上浮（translateY: -4px）
   - 阴影加深
   - 过渡时间：0.3s ease

2. **链接悬浮**

   - 下划线从左侧滑入
   - 颜色变化
   - 使用 CSS ::after 伪元素

3. **按钮悬浮**

   - 背景色变化
   - 轻微放大（scale: 1.05）
   - 阴影效果

4. **图片悬浮**
   - 轻微放大（scale: 1.05）
   - 过渡时间：0.3s ease

### 3.3 页面切换动效

1. **路由切换**

   - 淡入淡出（fade）
   - 或滑动切换（slide）

2. **加载状态**
   - Skeleton 加载动画
   - 进度条

### 3.4 表单动效

1. **输入框聚焦**

   - 边框颜色变化
   - 标签上移（Material Design 风格）

2. **提交按钮**
   - 加载状态：旋转动画
   - 成功：对勾动画
   - 失败：抖动动画

### 3.5 评论动效

1. **新评论添加**

   - 从下方滑入
   - 淡入效果

2. **回复展开**
   - 点击"查看更多回复"时，平滑展开全部回复
   - 使用高度动画，过渡时间 0.3s

## 九、埋点系统设计

### 9.1 埋点事件类型

#### 页面访问事件

- `page_view`: 页面浏览
  - 属性: page_type, page_id, referer, title
- `page_leave`: 页面离开
  - 属性: page_type, page_id, duration

#### 用户交互事件

- `click`: 点击事件
  - 属性: element_id, element_type, text, position
- `scroll`: 滚动事件
  - 属性: scroll_depth (滚动深度百分比)
- `form_submit`: 表单提交
  - 属性: form_id, form_type (comment/subscribe)

#### 内容相关事件

- `article_view`: 文章浏览
  - 属性: article_id, article_title, category
- `article_read_complete`: 文章阅读完成
  - 属性: article_id, read_duration
- `comment_submit`: 评论提交
  - 属性: post_id, comment_length

### 9.2 埋点 SDK 设计

#### 前端埋点 SDK

```typescript
// 初始化
Analytics.init({
  apiUrl: "https://api.example.com",
  userId: getUserId(), // 匿名用户ID
  enableDebug: false,
});

// 页面浏览
Analytics.trackPageView({
  pageType: "post",
  pageId: "123",
  title: "文章标题",
});

// 自定义事件
Analytics.track("click", {
  eventName: "header_nav_click",
  elementId: "nav-blog",
  text: "博客",
});

// 文章阅读进度
Analytics.trackArticleProgress({
  articleId: "123",
  progress: 50, // 百分比
});
```

#### 数据上报策略

- **批量上报**: 收集事件后批量发送，减少请求
- **本地缓存**: 网络失败时缓存到 localStorage
- **节流防抖**: 滚动等高频事件使用节流
- **隐私保护**: 不收集敏感信息
